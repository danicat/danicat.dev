2025 年がバックミラーの中に消えていく中、私は（当然ながら）コーヒーを片手に座り、一体何が起きたのかを処理しようとしています。今年の言葉を一つ選ぶとしたら、それは **acceleration**（加速）でしょう。

業界の動きが速かっただけでなく、私自身の個人的、職業的な速度も、予想していなかった形でギアが変わりました。Google への入社から、AI 時代のエンジニアであることの意味の再定義まで、2025 年は深い「refactoring（リファクタリング）」の年でした。

## Google の章

4 月に、私は Google の Developer Relations チームに参加しました。それは何十年もかけて築き上げた夢であり、現実はその期待に応えるものでした。確かに、強烈な日々でした。イギリス、ヨーロッパ中を旅し、[ブラジル]({{< ref "/posts/20251029-six-months-at-google" >}}) の開発者コミュニティと再会する時間さえ見つけました。

しかし、移動した距離以上に、それは知的な旅でした。私は Google Cloud エコシステムに頭から飛び込み、[Vertex AI]({{< ref "/posts/20250605-vertex-ai-sdk-python" >}})、[MCP]({{< ref "/posts/20250817-hello-mcp-world" >}})、そして [Agent Development Kit (ADK)]({{< ref "/posts/20251020-diagnostic-agent-with-adk" >}}) で手を汚しました（実践経験を積みました）。それはソフトウェアの未来についての集中コースであり、私はその一瞬一瞬を愛していました。

## Vibe Coding の年

2024 年が AI と「遊ぶ」年だったとしたら、2025 年はそれを **受け入れる** 年でした。私は懐疑論者から ["vibe coding"]({{< ref "/posts/20251206-taming-vibe-coding" >}}) の完全なエバンジェリストへと変わりました。しかし、いいえ、あなたが考えている [**あの**](https://x.com/karpathy/status/1886192184808149383?lang=en) vibe coding ではありません。

私は、「コードが存在することを忘れる」というのは罠だと気づきました。代わりに、エンジニアリングの規律（明確な要件、コンテキスト管理、厳格なテスト）がこれまで以上に重要であることを学びました。ですから、私は異なる風味の vibe coding、つまり「規律ある vibe coding」を提唱しています。vibe coding というスラングを全く使うべきではないと主張する人もいるでしょうが、どうすればいいでしょう？この言葉は「キャッチー」なのです！

私はまた、開発者が「パイロット」として振る舞い、[Gemini CLI](https://geminicli.com/) や [Jules](https://jules.google/) のような AI エージェントの艦隊を操縦する、[モダンな開発者ワークフロー]({{< ref "/posts/20250714-developer-workflow" >}}) を構築しました。

この変化は単に哲学的なものではなく、実用的なものでした。私は自分自身の問題を解決するために、全く新しいツールを「vibe code（バイブコーディング）」しました。
*   **[GoDoctor]({{< ref "/posts/20250729-how-to-build-an-mcp-server-with-gemini-cli-and-go" >}}):** AI に、より良い Go コードの書き方を教えるための MCP サーバー。
*   **[Speedgrapher]({{< ref "/posts/20250805-introducing-speedgrapher" >}}):** テクニカルライティングの退屈な部分を自動化するためのツールキット。
*   **[AIDA]({{< ref "/posts/20251031-building-aida" >}}):** 私の「スタートレック」コンピューターの夢の実現。実際に会話ができる診断エージェントです。

## Hello, MCP World!

ツールについて言えば、2025 年は **Model Context Protocol (MCP)** がすべてを変えた年でした。AI にとっての「USB-C モーメント」と呼ぶ人もいます。私はそれを「AI のための HTTP/REST」と呼んでいます。私たちは過去 20 年間、システムをプログラムで呼び出せるように自動化を構築してきました。エンジニアリングの次のサイクルは、古いシステムを改修し、「エージェント的に（agenticly）」呼び出せる新しいシステムを構築することです。ついに、私たちのモデルを現実世界に接続するための標準的な方法ができました。

私はこの技術に大きく賭け、[GopherCon UK で基調講演]({{< ref "/posts/20250817-hello-mcp-world" >}}) を行い、独自のサーバーを構築する方法について広範囲に執筆しました。コミュニティがこの標準の周りに結集するのを見るのは、非常に心強いことでした。私たちはもはやボットとチャットしているだけではありません。私たちは物事を *実行する* システムを構築しているのです。

## 個人的な Refactoring

コードやカンファレンスだけではありませんでした。私は自分の物理的なハードウェアをリファクタリングする時間も取りました。今年は **20kg 以上** の減量に成功したと言えることを誇りに思います！私は今、私の成人生活全体の中で最高の体調にあります。協力的な環境で好きな仕事をすることは、存在する最高の健康プランであることがわかりました。

また、時間そのものについても多くの時間を費やして考えました。[なぜ年をとるにつれて時間が加速するように感じるのか]({{< ref "/posts/20250811-a-model-for-perceived-time" >}}) についての私の探求は、多くの皆さんの共感を呼びました。それは AI によってパワーアップされた、シャワー中のふとした考えから始まりました... 多くの人が AI は創造性に悪いと言いますが、私は、現実世界では探求するための時間、忍耐、または献身を持てないような思考やアイデアを探求するために、AI と遊ぶのが好きです。私にとって AI の力は、そうでなければやらないようなことを可能にしてくれることです（残念ながらまだ皿洗いはできませんが、アニメのロボットメイドを買うためには列の先頭に並ぶつもりです）。

## 2026 年に向けて

さて、次は何でしょう？

2025 年が **adoption**（採用）の年、つまりツールを学び、私たちのワークフローに適合させる年だったとしたら、2026 年は **kaizen**（改善）の年になると信じています。これらのツールがどのように振る舞うか、何が機能し、何が機能しないかについて学ぶべきことはたくさんあります。私たちは集合的に実験し、新しいことを試し、プロセスと学びを文書化しています。コード生成のためであれ、生活をシンプルにする完全な自律エージェントのためであれ、この技術を使用しながら、これらすべての教訓を統合し、より高い成熟度に到達する時が来ました。

私は **Personalized Software**（パーソナライズされたソフトウェア）についてさらに深く掘り下げるつもりです。特注のツールを構築するための参入障壁は崩壊しました。私たちは、問題があればアプリを探すのではなく、アプリを作る時代に入っています。私はこのフロンティアを開拓し続け、Speedgrapher のように私のニーズに正確に合わせたツールをさらに作成するだけでなく、皆さんにも同じことをする方法を教えていきたいと思っています。

また、ゲーム開発者としての側面を解放し、少なくとも 1 つの Game Jam に参加したいと思っています。今年は挑戦しましたが、仕事の他の重要な側面に追いつく必要があったため、ゲームを完成させることができませんでした。今は少し快適なポジションにいるので、少し背伸びをすれば、1 つか 2 つのゲームを世に出すことができると信じています。

この旅の一部でいてくれてありがとう。記事を読んだり、トークを見たり、私と一緒に vibe coding をしたりしてくれた皆さん、ありがとう！

新しいバイブス、よりクリーンなコード、そしてもっとたくさんの猫の写真でいっぱいの 2026 年に乾杯。 o/
